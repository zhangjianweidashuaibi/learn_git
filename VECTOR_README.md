"""
多模态RAG系统 - 向量化配置说明
"""

"""
====================================
核心设计思路
====================================

1. 模型选择：
   - 文本编码: paraphrase-multilingual-MiniLM-L12-v2 (支持中文)
   - 图像编码: clip-ViT-B-32 (CLIP模型，支持跨模态对齐)

2. 向量库选择：
   - Chroma: 轻量级、本地存储、易于集成

3. 跨模态实现：
   - 使用CLIP模型实现文本和图片在同一向量空间
   - 支持文本查询图片、图片查询文本

====================================
使用步骤
====================================

Step 1: 安装依赖
    pip install -r requirements.txt

Step 2: 运行向量化
    python multimodal_vector_store.py

Step 3: 在你的应用中使用向量库

====================================
高级配置
====================================

如果需要更好的模型，可以替换：

1. 更强的文本模型：
   - text_model_name = "paraphrase-multilingual-mpnet-base-v2"

2. 更强的图片模型：
   - image_model_name = "openai/clip-vit-large-patch14"

3. 自定义检索参数：
   - top_k: 调整返回结果数量
   - filters: 添加过滤条件（如只返回包含图片的块）

====================================
查询示例
====================================

# 文本查询
results = vector_store.search(
    query_text="如何安全使用冰箱？",
    top_k=5
)

# 图片查询
results = vector_store.search(
    query_image_path="query_image.png",
    top_k=5
)

# 带过滤条件的查询
results = vector_store.search(
    query_text="安装说明",
    top_k=5,
    filters={"has_images": True}  # 只返回包含图片的结果
)

====================================
注意事项
====================================

1. 首次运行会下载模型（约500MB-2GB）
2. 向量化时间取决于数据量和硬件
3. Chroma数据存储在指定目录，可持续使用
4. 如需重新向量化，会自动清理旧数据

====================================
下一步
====================================

向量化完成后，可以：
1. 集成到RAG问答系统
2. 添加重排序（rerank）提升精度
3. 添加混合检索（BM25 + 向量）
4. 实现缓存机制
"""

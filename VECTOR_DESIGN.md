# 支持长文本的多模态RAG向量化方案

## 问题分析

**原方案的问题**：
- CLIP文本编码器只支持77个token
- 你的文本块有1768个token → 会被严重截断
- 导致检索质量大幅下降

---

## 新方案：双重索引

```
┌─────────────────────────────────────────────────────────────┐
│                    双重索引架构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐         ┌─────────────────┐          │
│  │   文本索引       │         │   图片索引       │          │
│  │  (长文本模型)    │         │   (CLIP模型)     │          │
│  │                 │         │                 │          │
│  │  文本块 ✓       │         │  文本块 ✓       │          │
│  │  混合块 ✓       │         │  混合块 ✓       │          │
│  │  图片块 ✗       │         │  图片块 ✓       │          │
│  └────────┬────────┘         └────────┬────────┘          │
│           │                           │                    │
│           └──────────┬────────────────┘                    │
│                      ▼                                     │
│           ┌──────────────────┐                            │
│           │    结果融合       │                            │
│           │  (加权打分合并)   │                            │
│           └──────────────────┘                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 核心设计

### 1. 模型选择

| 组件 | 模型 | 支持长度 | 用途 |
|------|------|----------|------|
| 文本编码 | jina-embeddings-v3 | 8192 tokens | 编码长文本块 |
| 图片编码 | clip-ViT-B-32 | - | 编码图片 |
| 跨模态查询 | clip-ViT-B-32 | 77 tokens | 文本查询图片索引 |

### 2. 索引策略

每个块会添加到**两个索引**：

**对于文本块**：
- 文本索引：用长文本模型编码完整内容
- 图片索引：用CLIP编码（自动截断到77 tokens）

**对于图片块**：
- 文本索引：跳过（没有文本）
- 图片索引：用CLIP编码图片

**对于混合块**：
- 文本索引：用长文本模型编码（文本 + OCR + 视觉描述）
- 图片索引：用CLIP编码图片

### 3. 查询策略

**文本查询**：
```
用户文本
   ├─→ 文本索引检索（长文本模型）
   │   └─→ 结果 + 权重 0.7
   │
   └─→ 图片索引检索（CLIP文本编码）
       └─→ 结果 + 权重 0.3

融合 = 0.7 × 文本索引结果 + 0.3 × 图片索引结果
```

**图片查询**：
```
用户图片
   └─→ 图片索引检索（CLIP图片编码）
       └─→ 结果（权重 1.0）
```

## 使用示例

### 基础查询

```python
# 长文本查询
results = vector_store.search(
    query_text="冰箱的安全注意事项，特别是关于电源连接、接地、以及使用时需要避免的危险情况",
    top_k=5
)

# 短文本查询
results = vector_store.search(query_text="如何安装", top_k=5)

# 图片查询
results = vector_store.search(query_image_path="query.png", top_k=5)
```

### 调整权重

```python
# 更重视图片（视觉内容更重要）
results = vector_store.search(
    query_text="外观图",
    top_k=5,
    text_weight=0.4,
    image_weight=0.6
)

# 更重视文本（文字说明更重要）
results = vector_store.search(
    query_text="详细步骤",
    top_k=5,
    text_weight=0.8,
    image_weight=0.2
)
```

### 带过滤

```python
# 只返回包含图片的结果
results = vector_store.search(
    query_text="安装",
    top_k=5,
    filters={"has_images": True}
)

# 只返回混合块
results = vector_store.search(
    query_text="步骤说明",
    top_k=5,
    filters={"chunk_type": "mixed"}
)
```

## 模型选择对比

### 文本模型选项

| 模型 | 最大长度 | 维度 | 速度 | 质量 | 推荐 |
|------|----------|------|------|------|------|
| jina-embeddings-v3 | 8192 | 1024 | 快 | 高 | ✓ 默认 |
| bge-large-zh-v1.5 | 512 | 1024 | 快 | 高 | 纯中文 |
| m3e-large | 8192 | 1024 | 中 | 高 | 中文 |
| paraphrase-multilingual-mpnet-base-v2 | 512 | 768 | 快 | 中 | 轻量 |

### 图片模型选项

| 模型 | 维度 | 速度 | 质量 | 推荐 |
|------|------|------|------|------|
| clip-ViT-B-32 | 512 | 快 | 中 | ✓ 默认 |
| clip-ViT-L-14 | 768 | 中 | 高 | 更高质量 |

## 性能考虑

### 内存占用
- 文本模型（jina-embeddings-v3）：约2GB
- 图片模型（clip-ViT-B-32）：约500MB

### 向量化时间
- 取决于文本长度和数据量
- 长文本1768 tokens会自动分块处理
- 建议batch_size=50（已优化）

---

## 优势

✅ **支持长文本** - 最大8192 tokens，你的1768 tokens完全没问题
✅ **保留文本完整性** - 不会截断关键信息
✅ **跨模态检索** - 文本查询可以检索图片内容
✅ **灵活权重** - 可调整文本/图片权重
✅ **双索引优势** - 结合长文本检索和跨模态检索

---

## 注意事项

1. 首次运行会下载模型（约2.5GB）
2. 文本索引中纯图片块不会被索引（正常现象）
3. 查询时两个索引会同时检索并融合结果
4. 长文本查询主要依赖文本索引（更准确）
